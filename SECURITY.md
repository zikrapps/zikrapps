# Security Policy

If you discover a security issue in this website or in any of the Zikr Apps
(Tazkirah, Misbaha, Mehrab), please report it privately rather than opening
a public GitHub issue.

## How to report

Email **salam@zikrapps.com** with:

- A clear description of the issue.
- Steps to reproduce, if applicable.
- Any proof-of-concept code, screenshots, or HTTP requests.
- Your name or handle, if you'd like to be credited in the fix notes.

We will acknowledge your report within 3 business days and aim to provide a
remediation timeline within 10 business days for confirmed vulnerabilities.

## Scope

In scope:

- The marketing website at `zikrapps.com` and its subdomains.
- Source code in this repository.
- Beta APK binaries distributed via GitHub Releases.

Out of scope:

- Third-party services we integrate with (Cloudflare, Google Workspace,
  Formspree, GitHub) — please report directly to those providers.
- Issues that require physical access to a victim's device.
- Social engineering attacks against staff.

## Safe harbor

We will not pursue legal action against researchers who:

- Make a good-faith effort to avoid privacy violations, service disruption,
  and destruction of data.
- Give us a reasonable opportunity to fix the issue before public disclosure.
- Do not exfiltrate or retain user data beyond what is necessary to
  demonstrate the issue.

Thank you for helping keep Zikr Apps and our users safe.
